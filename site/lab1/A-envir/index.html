    附录A 实验环境安装

    写在前面

      实验室的电脑均已提前安装好实验环境。本附录仅供自行安装环境使用。

      本附录仅以WSL2为例介绍实验环境的安装方法，但实际也可参考本附录将RV32 GNU安装在现成的Linux环境中。

    1. 安装WSL2

      首先，确保已在BIOS中开启了CPU虚拟化功能。开启后，可在任务管理器中看到如图A-1所示的信息。

    图A-1 在任务管理器查看是否已开启CPU虚拟化功能

    关于开启CPU虚拟化

      不同型号电脑的BIOS可能不同，开启虚拟化的具体操作可能也不相同，请自行使用搜索引擎搜索或自行探索具体的开启方法。

      在任务栏搜索框搜索“PowerShell”，或在开始菜单找到PowerShell，右键选择 以管理员权限 打开，并在其中执行以下命令：

    • 命令1：
      dism /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
    • 命令2：
      dism /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

      命令较长，容易输入错误。建议点击代码块右侧的复制按钮以复制命令，并在PowerShell中单击鼠标右键粘贴后再执行。

    命令说明

      命令1用于开启Windows系统的Linux子系统功能（Windows Subsystem for Linux，WSL）。

      命令2用于开启Windows系统的虚拟机功能。若命令2执行出错，则可以尝试执行命令3：

    Enable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform -NoRestart

      上述2条命令执行成功后，重启 系统。

      再次 以管理员权限 打开PowerShell，执行命令wsl --set-default-version 2，从而将WSL的默认版本设置成WSL2。

      上述命令均执行成功后，下载并安装WSL2的Linux内核更新包 wsl_update_x64.msi 。

    2. 安装Linux发行版

      在任务栏搜索框搜索“Store”，或在开始菜单找到Microsoft Store。在Microsoft Store中搜索关键词“Linux”，选择你喜好的Linux发行版进行安装。

      若使用最新版本的Windows（内部版本号 20262），则也可在PowerShell中执行 wsl --install 命令。该命令将下载安装最新的Linux内核及Ubuntu发行版。

    3. 安装RV32 GNU

      本课程实验提供已经编译好的RV32 GNU工具包 rv32_gnu.tar.gz ，内含RV32编译器、Spike仿真器和riscv-pk代理内核。该工具包可以直接安装在Linux环境中使用。感兴趣的同学们也可以访问GitHub上的官方仓库riscv-collab/riscv-gnu-toolchain，并按照README文档自行编译安装。

      首先，需要将rv32_gnu.tar.gz文件拷贝到虚拟机中。

      通过快捷键 Win+E 打开文件资源管理器，然后在地址栏输入 \\wsl$ 并回车，或点击左下方的Linux，即可访问虚拟机文件系统，如图A-2所示。

    图A-2 访问虚拟机文件系统

      在图A-2中，每个文件夹均代表一个Linux发行版的文件系统，且文件夹与发行版同名。

      双击进入相应文件夹，然后再依次双击进入/home/XXX/的用户目录。将rv32_gnu.tar.gz拷贝到其中，如图A-3所示。

    图A-3 拷贝rv32_gnu.tar.gz至用户目录

      在开始菜单找到安装好的Linux发行版，单击启动之。

    如果已安装了其他发行版

      推荐使用WSL2发行版管理器 wsl2-distro-manager 对已安装的多个发行版进行管理。该工具具有可视化界面，不仅操作方便，而且体积小、解压即用。

      也可在PowerShell中执行wsl -l -v查看所有已安装的发行版，然后执行wsl -d <发行版名称>命令以启动相应的发行版，如图A-4所示。

    图A-4 查看并启动WSL中已安装的Linux发行版

      启动发行版后，在其终端执行解压命令：

    sudo tar -zxvf rv32_gnu.tar.gz -C /opt

      接着，需要添加环境变量。

      在Windows文件资源管理器中，进入发行版的用户目录，找到.bashrc文件，如图A-5所示。

    图A-5 找到.bashrc文件以添加环境变量

      双击打开.bashrc文件，在最后增加两行代码：

    1
2
    export PATH=$PATH:/opt/riscv_gnu/riscv32-unknown-elf/bin/
export PATH=$PATH:/opt/riscv_gnu/riscv64-unknown-elf/bin/

      保存文件后，回到Linux发行版的终端，执行source .bashrc命令，从而使环境变量生效。

      在终端中输入ris并按下 Tab 键，观察命令是否被自动补全。若自动补全，说明RV32的GNU环境已安装成功。

      为运行编译后的RV32程序，还需输入命令sudo apt install device-tree-compiler以安装设备树编译器。

      最后，对所安装的RV32交叉编译环境进行测试。

      编写HelloWorld程序，执行命令riscv32-unknown-elf-gcc helloworld.c对其进行编译，然后执行命令spike --isa=rv32g pk a.out对其进行仿真运行，如图A-6所示。

    图A-6 测试RV32交叉编译环境

      若编译C程序时出现“error while loading shared libraries: libmpc.so.3”的错误提示，则在终端执行sudo apt install libmpc3以安装所需的库即可。
