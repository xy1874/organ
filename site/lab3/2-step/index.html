module ICache(
    input  wire         cpu_clk,
    input  wire         cpu_rst,        // high active
    // Interface to CPU
    input  wire         inst_rreq,      // 来自CPU的取指请求
    input  wire [31:0]  inst_addr,      // 来自CPU的取指地址
    output reg          inst_valid,     // 输出给CPU的指令有效信号（读指令命中）
    output reg  [31:0]  inst_out,       // 输出给CPU的指令
    // Interface to Read Bus
    input  wire         mem_rrdy,       // 主存就绪信号（高电平表示主存可接收ICache的读请求）

    output reg  [31:0]  mem_raddr,      // 输出给主存的读地址
    input  wire         mem_rvalid,     // 来自主存的数据有效信号
    input  wire [`BLK_SIZE-1:0] mem_rdata   // 来自主存的读数据

    localparam IDLE  = 2'b00;

    localparam STAT1 = 2'b11;
    reg [1:0] state, nstat;

    注意事项

      若只完成必做题，除了ICache模块（ICache.v），不要修改SoC工程的其他任何文件！

      若完成必做题和附加题，除了ICache模块（ICache.v）和DCache模块（DCache.v），不要修改SoC工程的其他任何文件！

      （4）参照下一节的仿真调试说明，运行功能仿真并根据仿真波形完成调试；

      （5）测试并比较无ICache和有ICache时，SoC运行测试程序的时间，并记录ICache命中率；

    关闭ICache的方法

      将头文件defines.vh第2行代码的宏定义“#define ENABLE_ICACHE”注释掉即可。

      （6）反思实验过程，总结收获并整理遇到的问题及解决方案，撰写实验报告。

    2. 仿真调试说明

    2.1 顶层模块设置

      在Sources窗口下找到Simulation Souces，展开sim_1文件夹，将其内的soc_simu.v设置成仿真的顶层文件，如图3-1所示。

    图3-1 更改仿真的顶层模块为soc_simu.v

    2.2 测试程序设置

      在Sources窗口下依次找到并展开soc.v和axiram_wrap.v模块，双击打开存储主存数据的blk_mem_gen_0IP核，将Other Options下的初始文件更换成start.dump.coe，如图3-2所示。

    图3-2 打开Block Memory IP核的配置窗口，更改测试程序

      在图3-2中点击OK按钮，并在随后打开的Generate Output Products窗口中点击Skip。

    咦，IP核被锁住了，无法操作？

      本实验提供的模板工程是基于Vivado 2018.3构建的。使用更高版本的Vivado打开模板工程时，工程中的IP核将被锁住，无法直接修改。为此，我们需要升级工程中的IP核。

      具体操作是，在任意一个被锁住的IP核上右键，点击“Report IP Status”，然后在最下方的IP Status窗口中，点击Upgrade Selected按钮，随后在弹出的对话框中点击OK按钮，即可一键升级所有IP核，如图3-3所示。

    图3-3 一键升级所有IP核

    2.3 运行功能仿真

      完成上述两处修改后，运行功能仿真，并点击工具栏的“Run All”按钮，如图3-4所示。

    图3-4 点击“Run All”按钮以运行测试程序

      TestBench将自动运行直到通过测试或发现错误。如果波形窗口没有任何信号和波形，请首先确保当前查看的是不是soc_simu_behav.wcfg的波形配置文件，如图3-5所示。

    图3-5 仿真波形配置文件

      在图3-5所示的波形配置文件中，ICache_Interface下的信号是ICache模块的接口信号，hit信号是ICache内部的命中信号，而ifetch_cnt则是CPU的取指计数器。

      仿真调试时，请根据实际需要自行添加ICache模块内部的信号到波形窗口中。

      若TestBench运行数秒后，仿真波形中的信号仍未有变化，说明CPU未能从ICache中取出指令，此时应当仔细检查ICache逻辑。

      仿真时，TestBench将在控制台打印每次取指或访存的调试辅助信息，如图3-6所示。

    图3-6 TestBench将在控制台输出调试辅助信息

      成功通过测试程序后，TestBench将在控制台打印测试程序的仿真运行总时间、总取指数量、ICache命中率等信息，如图3-7所示。

    图3-7 通过测试时，控制台的提示信息
